### 1. 계획 (Plan)

- 작업 목표: Go Core에서 Gitea Webhook을 검증, 중복 방지, 원본 저장할 수 있는 초기 백엔드 기반을 구현한다.
- 프론트엔드 전제: 프론트엔드는 별도 브랜치에서 개발 후 병합 예정이므로, 백엔드는 REST/WebSocket 계약이 흔들리지 않도록 API 경로와 응답 형태를 안정적으로 유지한다.
- 초기 완료 기준:
  - Go Core 구조를 설정/HTTP/store 계층으로 분리.
  - `webhook_events` PostgreSQL migration 추가.
  - `POST /api/v1/integrations/gitea/webhooks` endpoint 구현.
  - `GITEA_WEBHOOK_SECRET` 기반 signature 검증 및 중복 dedupe key 처리.
  - 단위 테스트와 `cd backend-core && go test ./...` 통과.
