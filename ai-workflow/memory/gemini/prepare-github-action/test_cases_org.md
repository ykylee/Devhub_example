# Test Cases — 조직 관리 (Organization Management) 1차 완성 검증

- 문서 목적: M3 마일스톤의 핵심인 '조직 관리 1차 완성'을 검증하기 위한 E2E 테스트 케이스 리스트.
- 범위: 조직 단위(Unit)의 CRUD, 멤버/리더 배정, 계층 구조 시각화 및 편집.
- 대상 독자: E2E 작성자, QA, 프로젝트 리드.
- 최종 수정일: 2026-05-12
- 상태: draft (사용자 검토 대기)

---

## 1. 검증 대상 기능 (Feature Map)

| ID | 기능 | UI 경로 | 주요 액션 |
| --- | --- | --- | --- |
| **F-ORG-LIST** | 조직 목록 조회 | `/admin/settings/organization` (List View) | 검색, 필터링, 상세 진입 |
| **F-ORG-CRUD** | 조직 단위 CRUD | `/admin/settings/organization` | 신규 부서 생성, 이름/타입 변경, 부서 삭제 |
| **F-ORG-MEM** | 멤버 및 리더 관리 | `MemberManagementModal` | 부서원 추가/삭제, 리더 임명/해임 |
| **F-ORG-CHART** | 조직도 시각화 | `/admin/settings/organization` (Chart View) | 계층 구조 확인, 노드 위치 변경(Drag) |

## 2. 테스트 케이스 상세

### F-ORG-LIST: 조직 목록 조회 및 가독성

#### TC-ORG-LIST-01: 리스트 뷰 기본 노출 및 가독성 확인
- **단계**:
  1. `/admin/settings/organization` 진입.
  2. 기본 뷰가 "List View"임을 확인.
  3. 테이블 컬럼(Unit Name, Type, Leader, Members)이 정상적으로 표시되는지 확인.
- **기대 결과**: 카드 형태가 아닌 테이블 형태로 조직 목록이 명확히 노출됨.

#### TC-ORG-LIST-02: 조직 단위 검색 (Search)
- **단계**:
  1. 리스트 뷰 상단 검색창에 `Engineering` 입력.
  2. 이름에 "Engineering"이 포함된 행만 필터링되는지 확인.
- **기대 결과**: 검색어에 따른 즉각적인 리스트 필터링 동작.

---

### F-ORG-CRUD: 조직 단위 관리 (CRUD)

#### TC-ORG-UNIT-01: 신규 부서 생성 (Create)
- **단계**:
  1. 리스트 뷰 상단 "Create Unit" 버튼 클릭.
  2. 모달에서 Name: `New Team`, Type: `Team`, Parent: `Engineering` 선택 후 저장.
  3. 리스트에 `New Team` 행이 추가되었는지 확인.
- **기대 결과**: 신규 부서가 성공적으로 생성되고 목록에 반영됨.

#### TC-ORG-UNIT-02: 부서 정보 수정 (Update)
- **단계**:
  1. `New Team` 행의 "Manage" 또는 "Edit" 버튼 클릭.
  2. 부서 이름을 `Updated Team`으로 변경 후 저장.
  3. 리스트에서 이름이 변경되었는지 확인.
- **기대 결과**: 부서 정보 변경 사항이 영속화되어 반영됨.

#### TC-ORG-UNIT-03: 부서 삭제 (Delete)
- **단계**:
  1. `Updated Team`의 상세/관리 메뉴에서 "Delete Unit" 클릭.
  2. 삭제 확인 다이얼로그 승인.
  3. 리스트에서 해당 행이 사라졌는지 확인.
- **기대 결과**: 부서가 정상적으로 삭제됨 (하위 부서가 있는 경우의 제약 조건 확인 포함).

---

### F-ORG-MEM: 멤버 및 리더 배정

#### TC-ORG-MEM-01: 부서 멤버 추가 및 삭제
- **단계**:
  1. 특정 부서의 "Manage" 버튼 클릭하여 멤버 관리 모달 진입.
  2. 미배정 사용자 중 한 명을 추가.
  3. 기존 멤버 중 한 명을 제거 후 저장.
  4. 리스트 뷰의 "Members" 카운트가 변경되었는지 확인.
- **기대 결과**: 부서원 구성 변경이 정확히 반영됨.

#### TC-ORG-MEM-02: 부서 리더 임명 및 변경
- **단계**:
  1. 멤버 관리 모달에서 "Leader" 드롭다운을 통해 특정 멤버를 리더로 선택.
  2. 저장 후 리스트 뷰의 "Leader" 컬럼에 왕관 아이콘과 함께 이름이 노출되는지 확인.
  3. 리더를 다른 사람으로 변경하여 정상 교체되는지 확인.
- **기대 결과**: 리더 임명 정보가 UI 및 목록에 정확히 표시됨.

---

### F-ORG-CHART: 계층 구조 시각화

#### TC-ORG-CHART-01: 조직도 렌더링 및 계층 연결 확인
- **단계**:
  1. 뷰 스위처에서 "Org Chart" 선택.
  2. 루트 부서와 하위 부서들이 선으로 연결되어 표시되는지 확인.
- **기대 결과**: 실제 조직 계층 구조가 그래프 형태로 정확히 시각화됨.

#### TC-ORG-CHART-02: 노드 드래그 및 위치 저장 (시뮬레이션)
- **단계**:
  1. 특정 부서 노드를 드래그하여 위치 이동.
  2. 페이지 새로고침 후에도 노드 위치가 유지되는지 확인 (또는 Save 버튼 클릭 후 확인).
- **기대 결과**: 사용자가 조정한 조직도 레이아웃이 보존됨.

---

## 3. 결정 사항 및 고려 사항
- **권한 제어**: 조직 관리 CRUD는 `system_admin` 권한이 있는 사용자(예: charlie)만 수행 가능해야 함.
- **데이터 정합성**: 부서 삭제 시 해당 부서에 소속된 멤버들의 처리 정책(미배정 상태로 전환 등)을 E2E 시나리오에 포함.
- **Empty State**: 검색 결과가 없거나 조직이 하나도 없는 경우의 UI 대응 확인.
