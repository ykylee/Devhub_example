## TASK-014 도메인 정규화 sink 및 webhook 처리 경로 연결

- 상태: done
- 우선순위: high
- 담당: Codex
- 영향 문서/코드: `backend-core/internal/domain/`, `backend-core/internal/store/`, `backend-core/internal/httpapi/`, `backend-core/main.go`, `ai-workflow/memory/backend_development_roadmap.md`

### 1. 계획 (Plan)

- 작업 목표: fixture 기반 normalizer를 실제 PostgreSQL 저장소와 webhook 처리 경로에 연결한다.
- 범위:
  - 정규화 domain type과 sink interface를 순환 참조 없이 분리.
  - `PostgresStore`에 repository/user/issue/pull_request/ci_run upsert 구현.
  - `webhook_events` 상태 전이 메서드(`processed`, `failed`, `ignored`) 구현.
  - webhook 저장 성공 이후 optional processor 실행 경로 추가.

### 2. 실행 (Act)

- `backend-core/internal/domain/domain.go`를 추가해 정규화 결과 type과 sink interface를 분리했다.
- `backend-core/internal/store/postgres.go`에 domain upsert와 webhook 상태 전이 메서드를 추가했다.
- `backend-core/internal/httpapi`에 `WebhookEventProcessor` interface를 추가하고, webhook 저장 성공 이후 processor를 호출하도록 연결했다.
- `backend-core/main.go`에서 DB store가 설정된 경우 `normalize.Processor{Sink: pgStore}`를 router에 주입했다.
- `backend-core/internal/store/postgres_integration_test.go`를 추가해 홈랩 PostgreSQL 기반 통합 테스트를 환경 변수 opt-in 방식으로 구성했다.

### 3. 검증 (Validate)

- `cd backend-core && go test ./...` 통과.
- `DEVHUB_TEST_DB_URL=... go test ./internal/store -run TestPostgresStoreProcessesNormalizedWebhookEvent -count=1 -v` 통과.

### 4. 결과 (Result)

- 작업 결과: Phase 6 도메인 정규화 1차의 핵심 경로가 연결됐다. raw webhook은 저장 후 즉시 정규화 processor를 거쳐 도메인 테이블 upsert와 `processed|failed|ignored` 상태 전이를 수행할 수 있다.
