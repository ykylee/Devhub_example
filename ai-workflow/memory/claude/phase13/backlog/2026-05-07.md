### 1.1 계획 (Plan)

- 작업 목표: ADR-0001 결정 7종을 반영한 Hydra+Kratos PoC 환경을 구성하고 Next.js `/login` 에서 OIDC code flow round-trip 1회 성공.
- sub-step 분리:
  - (a) **사용자 수동, 샌드박스 외 터미널**: `go install github.com/ory/hydra/v2/cmd/hydra@vX.Y.Z`, `go install github.com/ory/kratos/cmd/kratos@vX.Y.Z`. 정확 버전은 1단계 시작 시 latest stable 결정.
  - (b) **AI 자동화**: DB schema 생성 SQL (`devhub` 안 `hydra`, `kratos`).
  - (c) **AI 자동화**: 설정 파일 작성 — `infra/idp/hydra.yaml`, `infra/idp/kratos.yaml`.
  - (d) **AI 자동화**: Kratos identity schema JSON (`traits.email`, `traits.display_name`, `metadata_public.user_id`).
  - (e) **AI 자동화**: DevHub OIDC client 등록 스크립트.
  - (f) **검증**: hydra/kratos migrate sql → hydra serve / kratos serve → Next.js `/login` round-trip 1회 성공.
- 완료 기준: (b)~(e) 산출물 작성 + (f) round-trip 1회 성공 로그.

- OIDC client: `client_id=devhub-frontend`, redirect URI=`http://localhost:3000/auth/callback`, **public client + PKCE** (no secret), `skip_consent=true`.
- 서명 키: PoC default JWGS 자동 생성, 운영은 별도 결정.

AI 자동화 산출물 (sub-step b~e) 완료:

> 2026-05-07 install 시도 기록:
> 1. `…/v2/cmd/hydra@latest` / `…/cmd/kratos@latest` → `module found but does not contain package` 에러. main 이 모듈 루트에 있어서 cmd path 가 잘못됨.
> 2. `…/v2@latest` / `…@latest` (모듈 루트) → `go.mod` `replace` 지시문 차단 (`The go.mod file for the module providing named packages contains one or more replace directives`). `go install` 은 이 프로젝트에 사실상 사용 불가.
> 3. → GitHub release Windows SQLite binary 다운로드로 전환. `infra/idp/scripts/install-binaries.ps1` 가 v26.2.0 (release-train, 두 프로젝트 공통) 의 `*-windows_sqlite_64bit.zip` 을 받아 `$env:USERPROFILE\go\bin` 에 배치. SQLite 변형은 CGO 없음 + embed migration 자산 포함.

(a) 완료 후 본 backlog 항목으로 복귀해 (b) DB schema apply → (c) hydra/kratos migrate sql → (d) kratos serve / hydra serve all (별도 창 2개) → (e) `register-devhub-client.ps1` 실행 → (f) round-trip 검증 진입.

(f) round-trip 검증은 frontend `/auth/login`, `/auth/consent`, `/auth/callback` 라우트 구현이 선행되어야 한다. 본 PoC 1단계 완료 후 별도 backlog 항목으로 분리 가능.
