- sub-step 분리:
  - (a) **사용자 수동, 샌드박스 외 터미널**: `go install github.com/ory/hydra/v2/cmd/hydra@vX.Y.Z`, `go install github.com/ory/kratos/cmd/kratos@vX.Y.Z`. 정확 버전은 1단계 시작 시 latest stable 결정.
  - (b) **AI 자동화**: DB schema 생성 SQL (`devhub` 안 `hydra`, `kratos`).
  - (c) **AI 자동화**: 설정 파일 작성 — `infra/idp/hydra.yaml`, `infra/idp/kratos.yaml`.
  - (d) **AI 자동화**: Kratos identity schema JSON (`traits.email`, `traits.display_name`, `metadata_public.user_id`).
  - (e) **AI 자동화**: DevHub OIDC client 등록 스크립트.
  - (f) **검증**: hydra/kratos migrate sql → hydra serve / kratos serve → Next.js `/login` round-trip 1회 성공.
- 완료 기준: (b)~(e) 산출물 작성 + (f) round-trip 1회 성공 로그.

### 1.3 결과 (Result)

PoC 1단계 sub-step (a)~(e) 완료. (f) round-trip 은 frontend `/auth/{login,callback,consent}` 라우트 구현이 선행되어야 하므로 별도 task 로 분리.

- (a) **binary 설치** ✅ — `infra/idp/scripts/install-binaries.ps1` 로 v26.2.0 SQLite binary 를 `$env:USERPROFILE\go\bin\` 에 배치. `hydra version` / `kratos version` 양쪽 v26.2.0 확인.
- (b) **DB schema** ✅ — `psql` 미설치 환경 대응으로 `backend-core/cmd/idp-apply-schemas/main.go` Go 헬퍼 신규 작성 (backend-core 의 pgx/v5 의존성 재사용 — 별도 모듈은 corp GoProxy 차단으로 신규 의존성 다운로드 불가). 실행 결과 `hydra`, `kratos` schema 생성 확인.
- (c) **설정** ✅ — `infra/idp/hydra.yaml` 검증 통과, Hydra `:4444 / :4445` 정상 listening. `infra/idp/kratos.yaml` 은 두 번 수정 후 통과:
    - cipher secret 길이를 32 byte 이하로 (`length must be <= 32, but got 44` 에러).
    - `identity.schemas[0].url` 를 cwd 기준 경로로 (`./infra/idp/identity.schema.json` — Kratos 가 process cwd 기준 으로 해석하므로 repo root 에서 launch 필수).
- (d) **identity schema** ✅ — `infra/idp/identity.schema.json` (traits.email + display_name) 로드 확인.
- (e) **OIDC client 등록** ✅ — `register-devhub-client.ps1` 를 admin REST API (`POST /admin/clients`) 호출로 재작성. Hydra v26 CLI 의 `hydra create oauth2-client` 는 `--client-id` 가 무시되어 UUID 자동 발급 → 재현 불가. REST API 로 `client_id=devhub-frontend` 명시 가능. 등록 결과:
    ```
    client_id: devhub-frontend
    grant_types: authorization_code, refresh_token
    response_types: code
    redirect_uris: http://localhost:3000/auth/callback
    scope: openid offline_access email profile
    token_endpoint_auth_method: none (public client + PKCE)
    skip_consent: true
    ```
- (f) **round-trip** ⏸ — frontend `/auth/{login,callback,consent}` 라우트가 없어 보류. P1 7~8단계와 frontend Phase 5 의 합류 시점에 진행.

#### 검증 endpoint
- Hydra OIDC discovery: <http://localhost:4444/.well-known/openid-configuration> → `issuer http://localhost:4444/`, `authorization_endpoint /oauth2/auth`, `token_endpoint /oauth2/token`, `jwks_uri /.well-known/jwks.json` 확인.
- Hydra admin health: <http://localhost:4445/health/ready> → `{"status":"ok"}`.
- Kratos public health: <http://localhost:4433/health/ready> → `{"status":"ok"}`.
- Kratos admin health: <http://localhost:4434/health/ready> → `{"status":"ok"}`.
