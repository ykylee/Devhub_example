5. e2e 또는 통합 테스트 1건 — 무토큰 / 만료 토큰 / 정상 토큰 3 케이스.

## 🔐 SEC-2 — Backend BearerTokenVerifier 미주입 + nil fallback

- 상태: planned
- 우선순위: high
- 책임자: backend-core
- 발견 경위: PR #12 종합 리뷰 (2026-05-08, claude code). Codex/self-review 에서는 다루지 않음.
- 위치:
  - `backend-core/internal/httpapi/auth.go` — `cfg.BearerTokenVerifier == nil` 분기에서 `X-Devhub-Auth: bearer_unverified` 헤더만 부착 후 통과.
  - `backend-core/main.go` — `RouterConfig` 생성 시 `BearerTokenVerifier` 필드를 전혀 채우지 않음. 즉 prod 빌드도 nil fallback 경로로 동작.
- 영향: Authorization: Bearer 헤더가 붙은 요청은 토큰 내용 무관하게 통과. organization / RBAC / commands 등 mutation API 가 사실상 무인증.
- 마감: SEC-1 과 동일 — Phase 5.2 종료 전.

### 완료 정의 (DoD)

1. `BearerTokenVerifier` 구현체 1종 이상 존재 (예: IdP introspection / JWT 검증). 위치 후보 `backend-core/internal/auth/`.
2. `main.go` 에서 `cfg` 또는 환경변수 기반으로 verifier 인스턴스 생성하여 `RouterConfig.BearerTokenVerifier` 에 주입.
3. `auth.go` 의 nil 분기를 제거하거나, dev 전용 환경변수(예: `DEVHUB_AUTH_DEV_FALLBACK=1`) 가 명시될 때만 허용. 기본 동작은 401.
4. config layer 에 fail-fast 가드 — prod 모드에서 verifier 가 nil 이면 startup 거부.
5. `auth_test.go` 에 prod 가드 테스트 케이스 추가 (env unset + verifier nil → fail-fast).
