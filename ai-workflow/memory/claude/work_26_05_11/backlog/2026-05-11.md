- 흐름 (DEC-1=B)
  1. **Header Sign Out** (사용자 능동):
     a. backend `POST /api/v1/auth/logout` body `{refresh_token: <from tokenStore>}` 호출 → Hydra refresh revoke
     b. Kratos `/self-service/logout/browser` 호출 → `{logout_token, logout_url}` 받음
     c. token-store clear + zustand clearActor
     d. `window.location.assign(logout_url)` (Kratos 가 cookie 파괴 후 `default_browser_return_url`(`/`) 로 복귀)
     e. (Hydra OIDC session 은 다음 OIDC 진입 시 `prompt=none` 안 통하면 자연스럽게 재인증)
  2. **`/auth/logout?logout_challenge=…`** (Hydra OIDC RP-initiated logout — 외부 client 가 트리거):
     a. backend `POST /api/v1/auth/logout` body `{logout_challenge}` 호출 → `{redirect_to}` 받음
     b. Kratos browser logout 동시 호출 (Hydra 와 Kratos 양쪽 다 종료)
     c. token-store clear
     d. `redirect_to` 가 있으면 그쪽으로, 없으면 `/`

- DoD
  1. Header Sign Out → 위 1 흐름. 재로그인 시 Hydra 가 새 prompt 요구 (`prompt=login` 없이도 Kratos session 없으므로 자연스럽게)
  2. `/auth/logout` 라우트 + `logout_challenge` 없을 때 `/login` redirect
  3. backend 호출 실패해도 frontend 는 token-store clear + Kratos logout 만큼은 수행 (best-effort)

  5. e2e (사용자 환경): 로그인 → Sign Out → `/login` 자동 진입 시 password 재입력 요구 PASS
