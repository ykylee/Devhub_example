### 9.2 실행 (Act)
- 7종 결정 결과 (ADR-0001 §8 인라인):
  1. **DB 분리**: `devhub` 단일 DB + `hydra`, `kratos` schema 분리. `?search_path=...` 적용.
  2. **외부 앱 신뢰 경계**: 단계적 — 1차 사내 first-party only (`skip_consent=true`), 후속 phase 에서 외부 SaaS client + consent UI.

  6. **Binary 획득 (사용자 결정)**: `go install …@vX.Y.Z` 를 **샌드박스 외 사용자 터미널**에서 수동 실행. AI 자동화는 binary 설치 미수행, 설정/스크립트/검증만 수행.
  7. **호스트 서비스**: PoC 1차 직접 실행, 운영 진입 시점 별도 결정.
- backend_development_roadmap.md P1 큐 갱신:
  - 1단계(PoC) 를 (a) 사용자 수동 binary 설치, (b) DB schema, (c) 설정 파일, (d) Kratos identity schema, (e) 실행 방식, (f) 검증 6개 sub-step 으로 분해.
  - 7단계(token 미들웨어) 에 "`X-Devhub-Actor` deprecation warning 시작" 명시.
  - 큐 끝에 MFA / 외부 SaaS / Gitea SSO 가 본 phase 범위 밖임을 footnote 로 명시.
