# 조직도(Org Chart) 인터랙션 및 UX 명세

조직도 내에서 조직 단위를 직관적으로 관리하기 위한 고도화된 UX 명세입니다.

## 1. 조직 생성 흐름 (Creation Flow)

### 1.1 최상위 조직 생성
- **경로**: 차트 우측 상단 `[Add Unit]` 버튼.
- **동작**: 계층 구조상 부모가 없는 최상위 조직(Division 등)을 생성합니다.

### 1.2 하위 조직 생성 (Contextual Add)
- **경로**: 기존 조직 카드(Node) 내의 `[+]` 버튼.
- **동작**:
    1. 해당 노드를 부모(Parent)로 지정하여 새로운 하위 조직 노드를 생성합니다.
    2. 생성과 동시에 부모 노드와 새로운 자식 노드 사이에 연결선(Edge)이 자동으로 생성됩니다.
    3. 계층 규칙(예: 파트 아래에 팀 생성 불가)에 따른 타입 추천이 제공됩니다.

## 2. 조직 삭제 흐름 (Deletion Flow)
- **경로**: 조직 카드(Node) 내의 `[-]` 버튼.
- **동작**: 해당 조직 단위를 삭제하며, 하위 조직이 있는 경우 삭제 확인 팝업을 통해 연쇄 삭제 또는 상위 이관 여부를 확인합니다.

## 3. 정보 수정 및 확장 UI (Edit & Expand Flow)
- **경로**: 조직 카드(Node) 내의 `[Edit]` 버튼.
- **동작 (Card Expansion)**:
    - 카드가 고정된 크기에서 **수정 모드용 확장 UI**로 애니메이션과 함께 커집니다.
    - 확장된 영역에서 조직명, 리더 임명, 조직 설명 등을 직접 수정할 수 있습니다.
    - 수정 완료 시 다시 일반 카드 크기로 축소됩니다.

## 4. 계층 간 자동 정렬 (Hierarchical Alignment)
- **기능**: 자유로운 배치가 가능하지만, 조직의 계층 구조를 한눈에 파악할 수 있도록 노드들을 자동으로 정렬하는 기능을 제공합니다.
- **동작**: `Auto Layout` 버튼 클릭 시, 부모-자식 관계를 기반으로 상단에서 하단으로(Top-to-Bottom) 노드 위치를 재계산하여 배치합니다.
- **정렬 기준**:
    - 같은 계층(Sibling) 노드 간 간격 일정 유지.
    - 부모 노드가 자식 노드들의 중앙에 위치하도록 배치.

## 5. 시각적 가이드 (Visual Cues)
- **호버 효과**: 카드 호버 시 `+`, `-`, `Edit` 버튼이 부드럽게 나타납니다.
- **연결선 애니메이션**: 새로운 조직 추가 시 연결선이 그려지는 애니메이션을 제공하여 관계 형성을 시각화합니다.
