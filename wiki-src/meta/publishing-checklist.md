# Wiki Publishing Checklist

- 문서 목적: `wiki-src/` 원고를 GitHub Wiki에 게시하기 전 점검 항목을 표준화한다.
- 범위: 콘텐츠 검수, 공개 범위 점검, 링크 점검, 게시 순서
- 대상 독자: 문서 작성자, 리뷰어, 릴리즈 담당자
- 상태: active
- 최종 수정일: 2026-05-10
- 관련 문서: [Publication Matrix](./publication-matrix.md), [Style Guide](./style-guide.md)

## 사전 점검

- [ ] 대상 문서가 A/B/C 분류에서 공개 가능한 범위인지 확인
- [ ] 내부 경로/민감 정보/브랜치 이력 문구 제거
- [ ] 역할/권한 정책이 최신 기준과 일치하는지 확인

## 링크 점검

- [ ] Home ↔ Sidebar ↔ 본문 간 링크 정상
- [ ] 문서 간 상대 링크 깨짐 없음
- [ ] 외부 링크(있다면) 접근 가능

## 게시 순서 (권장)

1. Home
2. Sidebar
3. Quickstart
4. Tech Stack
5. Column 001
6. Column 002
7. Column 003
8. Meta (Publication Matrix, Style Guide)

## 게시 후 점검

- [ ] Wiki Home에서 모든 핵심 페이지로 이동 가능
- [ ] 모바일/데스크톱에서 제목/목차 가독성 확인
- [ ] 문서 제목과 URL slug 일관성 확인
